## Общий поток загрузки

1. Фронт делает первый запрос **на меню**: `GET /api/lists/menus`.
2. В ответ приходит список пунктов меню. Каждый пункт меню содержит:
   - маршруты фронта (`page`) и API (`api`);
   - и **`options`** — подробные правила, как рендерить таблицу/формы/действия/фильтры для этого раздела.

## Меню: `/api/lists/menus`

### Пример ответа для пункта меню «Заявки»

```json
{
  "id": 15,
  "name": "Заявки",
  "shortname": "orderexecutor",
  "parent_id": 13,
  "is_root": null,
  "is_api": 2,
  "level": 2,
  "page": "/web/work/orderexecutor",
  "api": "/api/work/orderexecutor",
  "model": "Modules\\Work\\Models\\OrderExecutor",
  "options": {
    "columns": {
      "client_place": {
        "fields": ["executor_id", "phone"],
        "name": "Исполнитель",
        "type": "double-fields"
      },
      "sum": {
        "fields": ["sum", "price_executor"],
        "name": "Сумма",
        "type": "double-fields"
      },
      "work_hours": {
        "fields": ["hours_executor", "hours"],
        "name": "Отработано часов",
        "type": "count"
      },
      "payStatus": {}
    },
    "fields": {
      "order_id": {
        "name": "Заказ",
        "type": "lookup",
        "field_model": "/Modules/Work/Models/Order",
        "lookup_api": "/api/work/order",
        "lookup_id": "id",
        "lookup_name": "name",
        "field_items": "orders",
        "field_prop": "order",
        "field_mode": "create,edit,delete"
      }
    },
    "actions": {
      "approve": {
        "name": "Выплатить",
        "icon": "uil-check",
        "theme": "button_stroke",
        "url": "/api/work/orderexecutor/{id}/status?status=1",
        "type": "ajaxBtn",
        "show_if": { "field": "workStatus", "op": "=", "val": 2 }
      }
    }
  },
  "icon": "uil uil-user",
  "status": 1
}
```

> [!NOTE]
> `model`, `is_root`, `is_api`, `resource` могут присутствовать в ответе, но по текущему описанию не обязательны для рендера.

### Схема объекта меню

| Поле | Назначение | Примечание |
|---|---|---|
| `name` | Название пункта меню |  |
| `shortname` | Короткое название (англ., одно слово) | Уникальный идентификатор |
| `parent_id` | ID родителя | Для 1-го уровня меню = `0` |
| `level` | Уровень вложенности |  |
| `page` | URL страницы фронта |  |
| `api` | Базовый URL API для данных |  |
| `options` | Правила рендера раздела | См. ниже |
| `icon` | Иконка из иконочного шрифта |  |
| `status` | Видимость пункта меню | `1` — показывать, `2` — скрыть |

## Как рендерится раздел

Визуально любой раздел — это один общий шаблон страницы:
`pages/web/[page]/[tab]/index.vue`

Шаблон рисует каркас (меню, хедер/футер, таблица, фильтры, пагинация), а **конкретный состав колонок/полей/действий** определяется через `options`.

> Скриншоты примеров интерфейса и форм находятся в каталоге `assets/` (в архиве экспорта/документации).

## Опции раздела: `options`

`options` — это набор JSON-объектов, управляющих выводом таблицы, форм, кнопок и фильтров.

| Объект | Назначение | Обязательность | Поведение по умолчанию |
|---|---|---:|---|
| `fields` | Список полей (таблица + формы create/edit/show) | ✅ обязательно | — |
| `columns` | Компоновка колонок (например 2 поля в одной колонке) | опционально | `1 поле = 1 колонка` |
| `column_groups` | Группировка колонок под общим заголовком | опционально | не группируются |
| `actions` | Действия строки | опционально | 4 стандартных: `просмотр, редактирование, дублирование, удаление` |
| `buttons` | Доп. кнопки над таблицей | опционально | только `Добавить запись` |
| `filters` | Панель фильтров | опционально | если нет — панель фильтров не показывается |
| `dop_filters` | «Жёсткие» фильтры (всегда применяются) | опционально | не применяются |
| `order` | Сортировка по умолчанию | опционально | `{"date_id":"asc","client_id":"desc"}` |
| `group` | Группировка записей в таблице | опционально | не группируются |
| `restrictions` | Ограничения отображения | опционально | не применяются |

---

### `fields` — поля

`fields` описывает поля сущности: как выводить в таблице и формах, как выбирать значения, какие есть вычисления и т.п.

#### Общие свойства поля (для любого `type`)

| Свойство | Назначение | Обязательность | По умолчанию | Примечание |
|---|---|---:|---|---|
| `<fieldKey>` | Ключ поля (обычно имя поля в БД) | ✅ | — | Это **имя объекта** внутри `fields` |
| `name` | Отображаемое название | ✅ | — |  |
| `type` | Тип/компонент поля | ✅ | — | Опции по типам — в отдельном разделе ниже |
| `field_mode` | Где показывать поле | опц. | `index,create,edit,show` | `index`, `create`, `edit`, `show`, `itog` |
| `fieldname` | «Фактическое имя» поля, если не берём `<fieldKey>` | опц. | — | Например `executor.phone` |
| `is_linked` | Значение берётся из связанного объекта через точку | опц. | `false` | `true` — значение рассчитывается из вложенного объекта |
| `db_type` | Тип поля в БД | опц. | — | Например: `string`, `integer`, `date`, `jsonb` |
| `validation` | Валидация для create/update | опц. | — | `rules.create`, `rules.update`, `messages.*` |
| `is_filter_need` | Нужно ли поле для фильтров | опц. | — | `true/false` |
| `column_group` | ID группы из `column_groups` | опц. | — | Для объединения колонок под общим заголовком |
| `title` | Управление заголовком колонки | опц. | — | `icon` — вместо текста показывать иконку |
#### Итоговая строка (`field_mode = itog`)

| Свойство | Назначение | Значения |
|---|---|---|
| `itog` | Операция в итоговой строке | `sum` (сумма), `cnt` (кол-во) |

---

### `columns` — колонки

`columns` нужен, когда используется нестандартная компоновка колонок (например, 2 поля друг под другом в 1 колонке).

```json
"columns": {
  "client_place": {
    "fields": ["executor_id", "phone"],
    "name": "Исполнитель",
    "type": "double-fields"
  },
  "payStatus": {}
}
```

| Свойство | Назначение | Обяз. | Примечание |
|---|---|---:|---|
| `<columnKey>` | Ключ колонки | ✅ |  |
| `fields` | Список полей из `fields` | ✅ |  |
| `name` | Название колонки | ✅ |  |
| `type` | Компонент отображения колонки | ✅ | Например `double-fields`, `count` |

> [!IMPORTANT]
> Если `columns` используется — **должны быть определены все колонки**.  
> Колонка может ссылаться напрямую на поле (тогда описывается как `{}` и ключ совпадает с именем поля).

---

### `actions` — действия строки

Если `actions` не указаны, используются 4 стандартных действия:
`просмотр`, `редактирование`, `дублирование`, `удаление`.

#### Пример `actions`

```json
{
  "approve": {
    "name": "Назначить",
    "icon": "uil-check",
    "url": "/api/work/orderexecutor/{id}/status?status=1",
    "type": "ajaxBtn",
    "show_if": { "field": "payStatus", "op": "!=", "val": 1 }
  },
  "places": {
    "name": "Места работы",
    "icon": "uil-bag",
    "url": "/web/clients/place/from/client_id/{id}",
    "type": "url"
  },
  "bids": {
    "name": "Исполнители на заказе",
    "icon": "uil-popup",
    "modalId": "#modal-performer-order",
    "type": "popup",
    "popup": { "name": "Исполнители", "page": "OrderExecutor" },
    "default": true
  }
}
```

| Свойство | Назначение | Обяз. | Поведение |
|---|---|---:|---|
| `name` | Текст кнопки | ✅ |  |
| `icon` | Иконка | опц. |  |
| `type` | Тип действия | ✅ | `popup`, `url`, `ajaxBtn` |
| `url` | URL перехода/запроса | опц. | Для `url`/`ajaxBtn`; поддерживает `{field}` |
| `popup` | Описание попапа | опц. | `name` + `page` |
| `default` | Открывать по клику на строку | опц. | `true` — открывать по клику |
| `show_if` | Условие показа | опц. | см. ниже |

`show_if`:

```json
{ "field": "payStatus", "op": "!=", "val": 1 }
```

### `buttons` — кнопки над таблицей

Определяются идентично `actions`, но отображаются сверху над таблицей рядом с кнопкой «Добавить запись».

---

### `filters` и `dop_filters` — фильтры

`filters` — фильтры, которые пользователь может выбирать в UI.  
`dop_filters` — «жёсткие» фильтры: **применяются всегда**, не показываются и не могут быть отменены.

#### Пример `filters`

```json
{
  "client_id": {
    "name": "Клиенты",
    "type": "multi-lookup",
    "kind": "external",
    "data": [{ "field": "client_id", "op": "in" }]
  },
  "date_order": {
    "name": "Дата заказа",
    "type": "text",
    "data": [{ "field": "date_order", "op": "between" }]
  },
  "executors": {
    "name": "Исполнители",
    "type": "multi-checkbox",
    "data": [{ "field": "status", "op": "in" }],
    "fields": {
      "executors_selected": { "name": "Подобрано", "type": "checkbox" }
    }
  }
}
```

---

### `order` — сортировка по умолчанию

Если не задано:

```json
{ "date_id": "asc", "client_id": "desc" }
```

---

### `group` — группировка записей

Позволяет группировать строки в таблице по значению определенного поля.

```json
"group": {
  "field": "report_date"
}
```

| Свойство | Назначение | Обяз. | Примечание |
|---|---|---:|---|
| `field` | Ключ поля для группировки | ✅ | Поле должно присутствовать в `fields` |

---

### `column_groups` — группировка колонок

Позволяет объединять несколько колонок под общим заголовком (header grouping).

```json
"column_groups": [
  {"id": 1, "name": "Продажа", "headerHozAlign": "center"},
  {"id": 2, "name": "Регистрация", "headerHozAlign": "center"},
  {"id": 3, "name": "", "headerHozAlign": "center"}
]
```

| Свойство | Назначение | Обяз. | Примечание |
|---|---|---:|---|
| `id` | Уникальный ID группы | ✅ | Используется в `fields[key].column_group` |
| `name` | Заголовок группы | ✅ | Может быть пустой строкой |
| `headerHozAlign` | Выравнивание заголовка | опц. | `left`, `center`, `right` |

---

### `restrictions` — ограничения

Пример:

```json
{
  "by_user": {
    "api": "/api/work/userclientrestriction",
    "user_id": "user_id",
    "restrict_id": "client_id",
    "restrict_name": "client_id"
  }
}
```

## Типы полей и их опции

Ниже — параметры, характерные **для конкретного** `type` в `options.fields`.

> Общие свойства (`name`, `type`, `field_mode`, `fieldname`, `is_linked`, `db_type`, `validation`, `is_filter_need`, `title` и т.п.) описаны в разделе `fields — поля`.

---

### Lookup-режим для любого контрола

Любой `type` может работать как lookup-поле (то есть получать отображаемое значение из справочника), если включить lookup-режим.

**Опции lookup-режима:**

| Свойство | Назначение | Обяз. | По умолчанию |
|---|---|---:|---|
| `is_lookup` | Включить lookup-режим | ✅ | — |
| `lookup_api` | URL API справочника | ✅ | — |
| `lookup_id` | Поле сопоставления в справочнике | опц. | `id` |
| `lookup_name` | Отображаемое поле из справочника | опц. | `name` |

> Устаревшие поля (не описываем): `field_model`, `field_prop`.

---

### `text`

Текстовое поле (в таблице и/или форме).

**Опции:**

| Свойство | Назначение | По умолчанию |
|---|---|---|
| `field_placeholder` | Плейсхолдер в форме | пусто |
| `format_view` | Формат отображения (например, дата/маска) | — |
| `modifier` | Представление текстового значения | — |
| `modifier_field` | Применить `modifier` не к текущему полю, а к другому | — |
| `template` | Шаблон ссылки (для `modifier: link`) | — |
| `target` | Target для ссылки | — |
| `icon` | Отобразить значение как иконку | — |
| `field_id` | Идентификатор поля для UI | — |

#### Значения `modifier`

- `link` — вывести значение как ссылку  
- `flag` — вывести значение как флаг. Флаги ищутся в ``assets/images/flags/``. Файл с флагом должен иметь расширение .svg  
- `icon` — вывести значение как иконку  
- `copy` — рядом со значением показать иконку копирования в буфер обмена  
- `pre` — вывести значение внутри `<pre>…</pre>`  
- `time` — отобразить значение как время `HH:mm`  
- `time_ago` — отобразить “неточное время” (например: `3 часа назад`)

#### `modifier_field`

Если указан `modifier_field`, модификатор применяется **не к текущему полю**, а к полю, указанному в `modifier_field`.

#### `template`

Используется вместе с `modifier: link` для построения ссылки по шаблону.  
Внутри `template` можно использовать подстановки `{fieldName}` — это имена других полей записи.

#### `icon_path`

Используется вместе с `modifier: icon` для указания пути поиска иконки.  
По умолчанию, если параметр не указан, используется ``assets/images/icons/``. Файл с иконкой должен иметь расширение .png

### `status`

Поле-статус. В таблице и карточке отображается как цветная таблетка-бейдж, в форме (create/edit) — как `<select>`.

**Опции:**

| Свойство | Назначение | По умолчанию |
|---|---|---|
| `field_items` | Список вариантов статуса | `{0:"Неизвестно", 1:"Активно", 2:"Выключено", 3:"В процессе"}` |

---

#### Форматы `field_items`

`field_items` принимается в двух форматах — движок нормализует их одинаково.

**1. Массив объектов** (рекомендуемый формат)

```php
'field_items' => [
    ['id' => 0, 'name' => 'Неизвестно'],
    ['id' => 1, 'name' => 'Активно'],
    ['id' => 2, 'name' => 'Выключено'],
]
```

**2. Словарь** `{id => label}`

```php
'field_items' => [
    '0' => 'Неизвестно',
    '1' => 'Активно',
    '2' => 'Выключено',
]
```

---

#### Цвет бейджа

Цвет можно задать двумя способами (оба работают в формате массива объектов):

**Вариант A — свойство `color`** (только фон, текст всегда белый):

```php
['id' => 1, 'name' => 'Активно', 'color' => '#22c55e']
```

**Вариант B — свойство `style`** (фон + цвет текста):

```php
[
    'id' => 1,
    'name' => '🔥 Срочная',
    'style' => [
        'backgroundColor' => '#FCE0AD',
        'color'           => '#FF7700',
    ],
]
```

Если ни `color`, ни `style.backgroundColor` не указаны — цвет берётся из встроенной палитры по числовому значению `id`:

| id % 8 | Цвет фона | Назначение |
|---:|---|---|
| 0 | `#6b7280` | серый |
| 1 | `#22c55e` | зелёный |
| 2 | `#ef4444` | красный |
| 3 | `#f59e0b` | янтарный |
| 4 | `#3b82f6` | синий |
| 5 | `#8b5cf6` | фиолетовый |
| 6 | `#06b6d4` | голубой |
| 7 | `#ec4899` | розовый |

---

#### Значения по умолчанию (без `field_items`)

Если `field_items` не задан, используются встроенные метки:

| id | Метка |
|---:|---|
| 0 | Неизвестно |
| 1 | Активно |
| 2 | Выключено |
| 3 | В процессе |

---

#### Внешний вид бейджа

Бейдж рендерится как `<span>` с inline-стилями:

```
display: inline-block
padding: 2px 10px
border-radius: 9999px  (таблетка)
font-size: 11px
font-weight: 600
```

#### Полный пример

```php
'status' => [
    'name'       => 'Статус задачи',
    'control'    => 'status',
    'field_mode' => 'index,create,edit,show',
    'db_type'    => 'integer',
    'is_lookup'  => false,
    'field_items' => [
        [
            'id'    => 0,
            'name'  => 'Новая',
            'style' => ['backgroundColor' => '#6b7280', 'color' => '#fff'],
        ],
        [
            'id'    => 1,
            'name'  => 'В работе',
            'style' => ['backgroundColor' => '#3b82f6', 'color' => '#fff'],
        ],
        [
            'id'    => 2,
            'name'  => 'Выполнена',
            'style' => ['backgroundColor' => '#22c55e', 'color' => '#fff'],
        ],
        [
            'id'    => 3,
            'name'  => 'Отменена',
            'style' => ['backgroundColor' => '#ef4444', 'color' => '#fff'],
        ],
    ],
],
```

## Подстановка значений в URL

В `url` разрешены плейсхолдеры вида `{field}` — вместо них подставляется значение **любого поля текущей записи**.

Пример:

```json
"url": "/api/work/orderexecutor/{id}/status?status=1"
```

---

## Спец-режим `/from/{field}/{id}`

Если добавить конструкцию:

`/from/{field}/{id}`

к **любой** странице фронта, то переход будет выполнен на страницу **с жёсткой фильтрацией** по полю `field`.

Пример:

`/web/clients/place/from/client_id/{id}`

---

## Примеры

### 1) Минимальный раздел (таблица из `fields`)

```json
{
  "fields": {
    "id": { "name": "ID", "type": "text", "field_mode": "index,show" },
    "name": { "name": "Название", "type": "text", "field_mode": "index,create,edit,show" }
  }
}
```

### 2) Два поля в одной колонке

```json
{
  "columns": {
    "title": { "fields": ["last_name", "first_name"], "name": "ФИО", "type": "double-fields" }
  }
}
```

### 3) Кнопка, доступная по условию

```json
{
  "actions": {
    "approve": {
      "name": "Подтвердить",
      "type": "ajaxBtn",
      "url": "/api/work/item/{id}/approve",
      "show_if": { "field": "status", "op": "=", "val": 1 }
    }
  }
}
```
